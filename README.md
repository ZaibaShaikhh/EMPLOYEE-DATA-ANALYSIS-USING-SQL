# EMPLOYEE-DATA-ANALYSIS-USING-SQL
Employee Data Analysis (SQL Project)
📊 Project Overview

This project focuses on analyzing employee, customer, agent, and order data using SQL queries. It demonstrates how structured data can be extracted, combined, and analyzed to generate meaningful business insights.

The project is presented in a PowerPoint format, showcasing SQL queries along with their explanations and insights.

🎯 Objectives
Perform data analysis using SQL queries
Understand relationships between multiple tables (Customer, Agents, Orders)
Extract meaningful business insights from raw data
Demonstrate use of joins, aggregations, and subqueries
🛠️ Tools & Technologies Used
SQL (Structured Query Language)
Relational Database Concepts
Microsoft PowerPoint (.pptx) for presentation
🗂️ Database Structure

The project uses the following tables:

Customer Table
Customer ID, Name, City, Country
Outstanding Amount, Payment Amount
Agents Table
Agent Code, Agent Name
Working Area
Orders Table
Order Number, Order Date
Order Amount, Advance Amount
Customer Code, Agent Code
🔍 Key SQL Operations Performed
🔗 Joins
INNER JOIN to combine customer and agent data
LEFT JOIN to include customers without orders
📊 Aggregations
COUNT → Number of orders per agent
SUM → Total order amount and outstanding balances
🔎 Filtering
WHERE clause for conditions (city, amount, dates)
HAVING clause for grouped data filtering
🧠 Subqueries
Finding highest order amount
Identifying customers with multiple orders
📈 Key Insights
Certain agents handle significantly higher order volumes
Some customers maintain high outstanding balances despite payments
City-wise analysis shows variation in customer distribution and financial activity
High-value orders and frequent customers can be identified easily using SQL
📷 Presentation Highlights
Step-by-step SQL query explanations
Business insights for each query
Clean and structured query breakdowns
